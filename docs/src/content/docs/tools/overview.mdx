## Tool Naming Convention

All Fizzy MCP tools follow a consistent naming pattern:

```
fizzy_{action}_{resource}
```

- **action**: `list`, `get`, `create`, `update`, `delete`, or specific verbs like `close`, `triage`
- **resource**: `board`, `card`, `comment`, `column`, `tag`, `user`

## Common Patterns

### Listing Resources

List tools accept optional filter parameters:

```
fizzy_list_cards({
  board_ids: ["abc123"],
  tag_ids: ["def456"],
  indexed_by: "golden",
  sorted_by: "newest"
})
```

### Getting Single Resources

Get tools require an identifier:

```
fizzy_get_card({ card_number: 42 })
fizzy_get_board({ board_id: "abc123" })
```

### Modifying Resources

Create/update tools accept content parameters:

```
fizzy_create_card({
  board_id: "abc123",
  title: "New feature",
  description: "Details here..."
})
```

## Error Handling

All tools return consistent error structures:

```
Error: Card not found
Code: NOT_FOUND_ERROR
```

See [Error Handling](/api/errors/) for details.
