# Config Card Reference

The config card is a special card on your board that links it to a GitHub repository and configures Vibe Coding behavior.

## What is the Config Card?

Every vibe-enabled board needs exactly one config card. It's a regular Fizzy card with:
- Title containing `config` (e.g., "Vibe Config", "AI Configuration")
- Tag: `#ai-config`
- Description: YAML configuration

The config card can live in any column — it's identified by its tag, not its position.

## Basic Configuration

Minimal config card:

```yaml
repository: owner/repo-name
```

This is all you need to get started. Vibe Coding will use sensible defaults for everything else.

## Full Configuration

All available options with their defaults:

```yaml
# Required: GitHub repository in owner/repo format
repository: owner/repo-name

# Branch to create PRs against (default: main)
default_branch: main

# Pattern for AI-created branches
# Available variables: {card_number}, {slug}, {timestamp}
branch_pattern: ai/{card_number}-{slug}

# Markdown template for PR descriptions
# Available variables: {card_title}, {card_number}, {card_url}, {card_description}
pr_template: |
  ## Summary
  
  Implements [{card_title}]({card_url}) (#{card_number})
  
  ## Changes
  
  {card_description}
  
  ---
  *Automatically generated by Fizzy Vibe Coding*

# Auto-assign PR to card assignees (default: true)
auto_assign_pr: true

# Maximum concurrent cards to process (default: 1)
max_concurrent: 1

# Labels to add to PRs (default: none)
pr_labels:
  - ai-generated
  - needs-review

# Draft PRs instead of ready for review (default: false)
draft_pr: false
```

## Configuration Options

### `repository` (required)

The GitHub repository in `owner/repo` format.

```yaml
repository: acme/my-application
```

**Validation**: Should match the git remote of the working directory where the AI processes cards.

### `default_branch`

The branch to create PRs against.

```yaml
default_branch: main
```

**Default**: `main`

Common alternatives: `master`, `develop`, `trunk`

### `branch_pattern`

Template for branch names created by the AI.

```yaml
branch_pattern: ai/{card_number}-{slug}
```

**Default**: `ai/{card_number}-{slug}`

**Variables**:
| Variable | Description | Example |
|----------|-------------|---------|
| `{card_number}` | Fizzy card number | `42` |
| `{slug}` | Slugified card title | `add-user-auth` |
| `{timestamp}` | Unix timestamp | `1699574400` |

**Examples**:
```yaml
# Results in: ai/42-add-user-auth
branch_pattern: ai/{card_number}-{slug}

# Results in: feature/fizzy-42
branch_pattern: feature/fizzy-{card_number}

# Results in: auto/1699574400-add-user-auth  
branch_pattern: auto/{timestamp}-{slug}
```

### `pr_template`

Markdown template for pull request descriptions.

```yaml
pr_template: |
  ## Summary
  
  Implements [{card_title}]({card_url})
  
  ## Card Description
  
  {card_description}
```

**Variables**:
| Variable | Description |
|----------|-------------|
| `{card_title}` | The card's title |
| `{card_number}` | The card's number |
| `{card_url}` | Full URL to the card in Fizzy |
| `{card_description}` | The card's description (markdown) |

### `auto_assign_pr`

Whether to automatically assign the PR to users assigned to the Fizzy card.

```yaml
auto_assign_pr: true
```

**Default**: `true`

Requires assignees to have linked GitHub accounts in Fizzy.

### `max_concurrent`

Maximum number of cards to process simultaneously.

```yaml
max_concurrent: 1
```

**Default**: `1`

::: warning
Values greater than 1 are experimental and may cause conflicts.
:::

### `pr_labels`

Labels to automatically add to created PRs.

```yaml
pr_labels:
  - ai-generated
  - needs-review
```

**Default**: None

Labels must exist in the GitHub repository.

### `draft_pr`

Create PRs as drafts instead of ready for review.

```yaml
draft_pr: true
```

**Default**: `false`

Useful if you want to review AI changes before requesting team review.

## Validation Rules

The AI validates the config card when processing work:

| Rule | Error |
|------|-------|
| Config card must exist | "No config card found on board" |
| Repository must be specified | "Missing required field: repository" |
| Repository format must be owner/repo or full URL | "Invalid repository format" |
| YAML must be valid | "Invalid YAML in config card" |

## One Config Per Board

Each board can have only one config card. If multiple cards are titled "vibe config", Vibe Coding uses the first one found.

To change configuration:
1. Edit the existing config card
2. Vibe Coding picks up changes on the next card pickup

## Example Configurations

### Minimal Setup

```yaml
repository: myorg/backend-api
```

### Team Project

```yaml
repository: myorg/backend-api
default_branch: develop
auto_assign_pr: true
pr_labels:
  - ai-generated
  - team-review
draft_pr: true
```

### Open Source Project

```yaml
repository: myorg/oss-library
default_branch: main
branch_pattern: contrib/ai-{card_number}
pr_template: |
  ## What
  
  {card_title}
  
  ## Why
  
  {card_description}
  
  ## Checklist
  
  - [ ] Tests pass
  - [ ] Documentation updated
  - [ ] Changelog entry added
  
  ---
  *Generated by [Fizzy Vibe Coding](https://fizzy.do)*
```
